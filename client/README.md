# client

Vite + React + TypeScript + Tailwind v4 PWA. Mobile-first dark UI inspired by Ring.

## Dev

```bash
npm install
npm run dev          # http://localhost:5173
```

The dev server proxies `/api` to `http://localhost:8000`. WebRTC video is fetched directly from `:8889` on the current host (MediaMTX WHEP endpoint), which means dev video only works when MediaMTX is running on the machine you're hitting (typically the Jetson).

## Build

```bash
npm run build        # outputs to dist/
npm run preview
```

The server can serve `dist/` as static files (see `server/app/main.py`).

## Tests

Vitest + Testing Library + jsdom. 219 tests across 19 files covering lib, components, and pages (count current as of iter-170; refreshed periodically — verify with `npm test` after changes).

```bash
npm test                 # one-shot run
npm run test:watch       # re-run on change
npm run test:coverage    # text + HTML coverage in coverage/
npm run typecheck
npm run lint             # ESLint flat config (react-hooks + jsx-a11y)
npm run lint:fix
```

Test layout mirrors source — `Foo.tsx` has `Foo.test.tsx` next to it. Conventions:

- **lib/** tests stub global APIs (`fetch`, `WebSocket`, `RTCPeerConnection`, `navigator.serviceWorker`, `Notification`) via `vi.stubGlobal` or `Object.defineProperty`.
- **component/** and **page/** tests use Testing Library queries (`getByRole`, `getByText`) — prefer accessibility-rooted queries over class names so refactors don't break tests.
- WebSocket / WebRTC behaviour is tested through small mock classes (see `src/lib/ws.test.ts` and `src/lib/webrtc.test.ts`) — they implement just enough of the API surface to exercise our handler logic.
- `src/test/setup.ts` registers `@testing-library/jest-dom` matchers and runs `cleanup()` after each test.

## Filesystem note

If this checkout lives on a filesystem that does not support symlinks (exFAT, FAT32, some NTFS mounts), `npm install` will fail to create the `node_modules/.bin` shims. Workarounds, in order of preference:

1. Move the project to an ext4 / btrfs / APFS / NTFS-with-links filesystem (e.g. `~/projects`) and use the standard `vite` / `vitest` scripts.
2. Stay where you are: `npm install --no-bin-links` and the `package.json` scripts (which invoke binaries through `node ./node_modules/.../bin.js`) will still work.

The current `package.json` uses the direct-node form so it works in either case. If you move to a normal filesystem, you can simplify the scripts back to `vite` / `vitest` / `tsc`.

## PWA

- Manifest + service worker generated by `vite-plugin-pwa` with `injectManifest` strategy
- Custom service worker at `src/sw.ts` handles `push` and `notificationclick` events
- Icons at `public/icon.svg` and `public/icon-maskable.svg`
- For production polish, replace SVGs with PNGs at 192/512/maskable sizes — Android Chrome accepts SVG but PNGs render more consistently across launchers. Generate with e.g. [pwa-asset-generator](https://github.com/elegantapp/pwa-asset-generator).

## Install on Android

1. Open the deployed URL in Chrome
2. Three-dot menu → "Add to Home Screen"
3. Launches fullscreen as "Home Camera"

## Source layout

```
src/
  main.tsx            entry
  App.tsx             routes + shell, ErrorBoundary per page
  index.css           Tailwind + theme
  sw.ts               service worker (push handling)
  components/
    BottomNav.tsx        tab bar
    ConnectionBanner.tsx top banner when WS is reconnecting / closed
    ErrorBoundary.tsx    last-resort wrapper around each page
    EventList.tsx        detection feed row
    LiveStats.tsx        compact two-row CPU / mem / worker readout
    Skeleton.tsx         loading-state placeholders
    Slider.tsx           range input with onChange / onCommit
    SnapshotPreview.tsx  full-screen modal after capture
    VideoTile.tsx        WHEP consumer + canvas overlay for boxes
  pages/
    Live.tsx          big tile, action buttons (Snapshot, Detect)
    Events.tsx        timeline of detections + person filter chips
    Settings.tsx      push toggle, detection sliders, schedule,
                      class chips, system health
  lib/
    api.ts            REST client; throws typed `HttpError` (iter-122)
    confirm.ts        Promise-based modal confirm dialog
    format.ts         shared formatUptime / formatAge helpers
    push.ts           Web Push subscribe / disable
    toast.tsx         ToastProvider + useToast hook
    types.ts          shared event/status types
    useStatus.ts      visibility-pause-aware /api/status poller
    webrtc.ts         WHEP connect helper (no STUN, recv-only video)
    ws.ts             event WebSocket (auto-reconnect, exp backoff)
```
